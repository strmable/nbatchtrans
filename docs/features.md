## 1. 프리픽스 추적 번역 완전성 검증

### 개요

각 번역 줄에 `[00001]|` 형식의 번호를 붙여 LLM에 전송하고, 번역 후 번호를 대조하여 누락된 줄을 감지합니다.

### 활성화

설정 탭 → **프리픽스 추적** 체크박스 (`enable_prefix_tracking`)

### 동작 방식

```
입력:
[00001]| 原文 첫 번째 줄
[00002]| 原文 두 번째 줄

출력:
[00001]| 번역된 첫 번째 줄
[00002]| 번역된 두 번째 줄
```

1. 각 비어있지 않은 줄에 `[NNNNN]|` 번호 부착 (청크 내에서 1부터 시작, 다음 청크에서 리셋)
2. `global_line_offset`으로 원본 파일 전체 기준 줄 번호 추적
3. 번역 후 번호 대조 → 누락 줄 감지
4. 프리픽스가 0개 감지되면 즉시 중단 (`BtgTranslationException`)

### 프롬프트 지시문

총 줄 수를 포함한 동적 지시문이 자동 삽입됩니다:

- 번호를 절대 건너뛰지 말 것
- 모든 N개 줄을 반드시 출력할 것
- 줄 병합/분리 금지
- 장 종결 마커(本章完, END 등) 이후에도 계속 번역할 것

## 2. 누락 줄 재번역

### 개요

1차 번역에서 누락된 줄을 별도로 재번역하여 보완합니다.

### 활성화

설정 탭 → **누락 줄 재번역** 체크박스 (`enable_prefix_missing_retranslate`)

> 프리픽스 추적(`enable_prefix_tracking`)이 활성화된 상태에서만 동작합니다.

### 처리 흐름

```
1차 번역 → 누락 줄 감지
  └→ 재번역 시도 (누락 줄만 별도 전송)
       ├→ 성공: 번역 결과 채움
       └→ 실패: 원문(중국어)으로 대체
```

- 재번역 없이 누락이 있는 경우: `@offset::N translate missing, original text "..."` 마커 삽입
- 재번역 후에도 누락인 경우: 원문 텍스트 그대로 삽입

## 3. 노이즈 패턴 처리

### 개요

소설 원문에 포함된 장 종결 표시(`(本章完)` 등)가 LLM의 조기 번역 종료를 유발합니다. 이를 방지하기 위해 LLM 입력에서 빈 줄로 교체하고, 출력 시 원문 그대로 복원합니다.

### 감지 패턴

| 패턴 | 예시 |
|------|------|
| `(本章完)` | 본장 완료 |
| `(完結)` | 완결 |
| `(完)` | 완 |
| `(終)` | 종 |
| `(End)` | End |
| `(THE END)` | THE END |
| `(fin)` | fin |

전각 괄호(`（）`)도 지원합니다.

### 동작

- LLM 입력: 해당 줄 → 빈 줄로 교체 (내용 완전 숨김)
- 출력: 원문 텍스트(`(本章完)` 등) 그대로 복원
- 번호(prefix)를 부착하지 않아 LLM이 조기 종료 신호로 인식하지 않음

## 4. 청크 품질 통계 로그

### 개요

각 청크 번역 완료 후 `[CHUNK_STATS]` 접두사로 시작하는 구조화된 통계 로그를 출력합니다.

### 로그 형식

```
[CHUNK_STATS] chunk=5 total=120 src=8432 trans=9156 ratio=1.09 api=12.3s total=14.1s
  src_lines=47 prefix_total=42 matched_1st=41 missing_1st=1 missing_rate=2.4%
  overflow=0 dup=0 retrans=True retrans_filled=1 final_missing=0 anomalies=[none]
```

### 주요 필드

| 필드 | 설명 |
|------|------|
| `chunk` / `total` | 현재 청크 번호 / 전체 청크 수 |
| `src` / `trans` | 원문 / 번역문 글자 수 |
| `ratio` | 번역문/원문 길이 비율 (정상: 0.8~3.0) |
| `missing_1st` | 1차 번역 누락 줄 수 |
| `retrans` | 누락 줄 재번역 실행 여부 |
| `final_missing` | 최종 누락 줄 수 |
| `anomalies` | 이상 징후 플래그 목록 |

## 5. GUI 로그 탭 색상 표시

### 개요

GUI 로그 탭에서 `[CHUNK_STATS]` 로그를 4단계 색상으로 구분하여 표시합니다. 다크/라이트 테마 자동 대응합니다.

### 4단계 색상 분류

| 단계 | 색상 | 조건 |
|------|------|------|
| 1단계 (정상) | 청록 | missing 없음, ratio 정상 |
| 2단계 (경고) | 황색 | 소량 missing (재번역으로 복구됨), anomaly 있음 |
| 3단계 (주의) | 주황 | `HIGH_MISSING`, `HIGH_RATIO`, ratio > 3.0 |
| 4단계 (오류) | 적색 | `final_missing > 0`, `DUPLICATE_PREFIX`, ratio < 1.1 |

### 기타 로그 색상

- `⚠️` 포함 메시지 → 경고 색상
- `🎯 전체 처리 완료` → 완료 색상
- `ERROR` 레벨 → 오류 색상

## 6. 검토 탭 재번역 병렬 실행

### 개요

검토 탭에서 여러 청크를 선택하여 재번역 시 최대 3개 청크를 동시에 번역합니다.

### 사용 방법

1. 검토 탭에서 Ctrl+클릭 또는 Shift+클릭으로 여러 청크 선택
2. **재번역** 버튼 클릭
3. 확인 다이얼로그에서 확인

### 실행 중 추가 기능

재번역 진행 중에 추가 청크를 선택하고 **큐에 추가** 버튼을 클릭하면 현재 배치 완료 후 자동으로 이어서 처리됩니다.

### 프리픽스 처리

재번역 시에도 배치 번역과 동일하게 프리픽스 추적이 적용됩니다:
- `enable_prefix_tracking=True`인 경우 프리픽스 부착 → LLM 번역 → 파싱 → 프리픽스 제거 후 저장
- `global_line_offset`은 해당 청크 이전 모든 청크의 줄 수 합산으로 자동 계산

## 7. API 오류 안정화

### 502 / 503 오류 처리

Gemini API에서 502 Bad Gateway 또는 503 Service Unavailable 응답 시 **30초 대기 후 자동 재시도**합니다.

감지 패턴:
- `502`, `503`
- `Bad Gateway`, `Service Unavailable`
- `UNAVAILABLE`
- `experiencing high demand`

### 재시도 정책

- 최대 재시도 횟수: 설정 탭 `max_retries` (기본값 적용)
- 서버 과부하 오류는 고정 30초 대기 후 재시도
- 기타 오류는 설정된 재시도 간격 적용

## 8. 용어집 동적 주입

### 개요

번역 시 용어집 JSON 파일을 자동으로 감지하여 프롬프트에 주입합니다.

### 활성화

설정 탭 → **용어집 동적 주입** 체크박스 (`enable_dynamic_glossary_injection`)

### 용어집 파일 위치

입력 파일과 같은 디렉토리에 `{파일명}_simple_glossary.json` 형식으로 저장합니다.

### 번역 규칙 (프롬프트 내 포함)

- 중국 인명은 **한자 독음(韓字 讀音)**으로 표기
  - 예: `毛澤東` → `모택동` (O), `마오쩌뚱` (X)
- 구조 마커(`***`, `---` 등)는 번역하지 않고 그대로 유지
- 연속되는 유사 문장도 절대 병합하지 않음

## 설정 키 참조

| 설정 키 | 기본값 | 설명 |
|---------|--------|------|
| `enable_prefix_tracking` | `false` | 프리픽스 추적 활성화 |
| `enable_prefix_missing_retranslate` | `false` | 누락 줄 재번역 활성화 |
| `enable_dynamic_glossary_injection` | `false` | 용어집 동적 주입 |
| `enable_context_injection` | `false` | 이전 청크 컨텍스트 주입 |
| `chunk_size` | `6000` | 청크 크기 (글자 수) |
| `max_retries` | `3` | API 최대 재시도 횟수 |
| `use_content_safety_retry` | `true` | 콘텐츠 안전 재시도 |
